# 【CS:APP 筆記】陣列與指標間的一些區別與可變大小陣列

## 陣列的記憶體位址

章節：3.8.2  
頁數：P. 257

當我宣告 `T A[N];` ，即一個型別為 T，有 N 個元素的陣列

此時 `&A == A`，寫個簡單的程式就可以驗證

但還是有一點區別的，`&A` 的型別為 `T (*)[N]`，但 `A` 的型別為 `T *`，而 `(A + 1)` 的型別也是 `T *`

也就是說意義上可以將 A 視為是一個指向陣列的指標，其位址(`&A`)為第一個元素的位址，其內容(`A`)為第一個元素的位址

如果是用 `T *B = (T *)malloc(sizeof(T) * N);` 來進行宣告的，`&B` 為該指標的位址，`B` 為其指向陣列的第一個元素的位址

在組合語言的層次，每一個 label 會對應一個記憶體位址(\&label)，如果把上面的 A 和 B 拿來看

A 對應的位址是 `&A`，`&A` 中的內容為 `A[0]`，而不是 `A`  
B 對應的位址是 `&B`，`&B` 中的內容為一個記憶體位址，指向一個陣列

也就是說應該是編譯器自動把 `A` 翻譯成 `&A`，因為它知道 A 是一個陣列

## 多維陣列與指標的取值

章節：3.8.5  
頁數：P. 263

``` c
#include <stdio.h>
#include <malloc.h>
#define M 5
#define N 4
 
int main() {
    int *A = (int *)calloc(M * N, sizeof(int));
    int B[M][N] = {{0}};
 
    printf("%d\n", B[2][3]);
 
    printf("%d\n", *(A + N * 2 + 3));
    /* printf("%d\n", *(B + N * 2 + 3)); */
    printf("%d\n", **(B + N * 2 + 3));
    printf("%d\n", *(*(B + 2) + 3));
 
    /* printf("%d\n", A[2][3]); */
    /* printf("%d\n", ((int [M][N])A)[2][3]); */
    /* printf("%d\n", ((int (*)[M][N])A)[2][3]); */
    printf("%d\n", ((int (*)[N])A)[2][3]);
 
    printf("%d\n", (*&B)[2][3]);
    printf("%d\n", (*((int (*)[M][N])&A))[2][3]);    // compile OK but answer is WRONG
}
```

這裡要注意一下，`B` 的型別是 `int (*)[N]` 而不是 `int [M][N]`，不過 `&B` 就是 `int (*)[M][N]`，具體參考前一部分。

章節：3.8.5  
頁數：P. 263
