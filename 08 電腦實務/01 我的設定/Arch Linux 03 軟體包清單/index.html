    Arch Linux 03 軟體包清單

    說明

    本文撰寫規則

    • 軟體包名稱沒有後綴的來自 Arch Linux 官方軟體庫，有 @A 後綴的來自 Arch User Repository (AUR)，有 @O 後綴的代表其它安裝方式。
    • 若軟體包名稱中有空格，則使用括號將其包起來，例如: (Package 1)
    • 標注 [未安裝] 表示目前電腦上未手動安裝的軟體包 (但有可能作為依賴被安裝)
    • 標注 [不須手動安裝] 表示會作為其他軟體包的依賴被安裝，不須手動安裝

    條目範本:

    更新注意事項

    • 更新前軟體包前，建議先更新鏡像伺服器列表。
    • 更新前記得閱讀 Arch Linux 首頁的 news (或訂閱 arch-announce)，確認是否有需要人工干預的更新。
      在更新重要軟體包 (如: kernel, xorg, systemd, glibc) 前，先進行備份，並到論壇看看是否有災情，不要在要使用電腦執行重要工作前更新。
      System maintenance - ArchWiki
    • 使用 AUR 所安裝的 Python 模組，需在 Python 更新後重新 makepkg(3.8 -> 3.9 才要，3.8.1 -> 3.8.2 不用)

    Pacman 指令速查

    • pacman 會保留所有下載過的軟體包作為快取，佔用很大儲存空間，可以執行 sudo pacman -Scc 清理全部，或執行 sudo paccache -r 保留最新的三個版本並清除其它更舊的版本。
    • 使用 sudo pacman -S --needed <package> 可以避免重新安裝已經安裝(且為最新版本)的軟體包。
    • 使用 sudo pacman -D --asdeps <package> 可以指定軟體包為 "作為其他軟體包的依賴安裝"， 使用 sudo pacman -D --asexplicit <package> 可以指定軟體包為 "單獨指定安裝"。
    • 使用 pacman -Qm 可以列出所有已安裝的，非 Arch Linux 官方軟體庫的軟體包 (如: AUR)。
    • 查看軟體包擁有的檔案
      • 已安裝的軟體包: pacman -Ql <package>
      • 未安裝的軟體包
        • 下載檔案資料庫: sudo pacman -Fyy (會下載到 /var/lib/pacman/sync/<repo>.files，裡面包含該 repo 的軟體包所擁有的檔案清單)
        • 查詢: pacman -Fl <package> (pacman -Sl <package> 沒用)
        • 刪除檔案資料庫: 刪除 /var/lib/pacman/sync/<repo>.files 即可
    • 查看 orphan packages: 執行 pacman -Qtdq (還可以安裝 pacman-log-orphans-hook@A (見下文))
    • Arch Linux Archive 提供官方編譯的舊版本軟體包，可供降級時使用 (詳細說明見該網頁)
      • 軟體包可下載後執行 sudo pacman -U <file> 安裝，亦可直接執行 sudo pacman -U <URL> 安裝 (後者會自動下載 <URL>.sig (detached OpenPGP signature) 並驗證軟體包)

    Linux Kernel Module

    • Dynamic Kernel Module Support (DKMS)
      • 軟體包: dkms 及已安裝的 kernel 的 header files (例如: linux 需要 linux-headers、linux-lts 需要 linux-lts-headers)
        • Arch Linux 已經預編譯了一些 kernel module，但只有給最新的 kernel (linux)
        • 若 (1) 不使用最新的 kernel (linux) 或 (2) 同時有安裝其他 kernel，就必須使用 dkms
        • 另外，Arch Linux 預編譯的 kernel module 通常會與 dkms 衝突，只能安裝該軟體包或 dkms 其中一個
      • 說明
        • 通常當 kernel 或某個 kernel module 更新後，pacman 會自動執行 DKMS 來重新建置 kernel module
        • 若需要手動重新建置，見 Dynamic Kernel Module Support - ArchWiki
        • 見 VirtualBox 一節

    網路

    • Netfilter framework

      • 軟體包: nftables iptables-nft
        • 依賴關係: iptables-nft 依賴於 nftables；iptables-nft 取代 iptables；iptables-nft 提供 iptables
        • base meta package 的 iproute2 和 systemd 依賴於 iptables，但於安裝 base 時，無法選擇使用 iptables-nft，所以要手動安裝
      • 設定
        • nftables.service、iptables.service 和 ip6tables.service 是 nftables 和 iptables/iptables-nft 自帶的簡單防火牆，若有使用其他防火牆管理工具 (如: UFW)，就不需要啟動
      • 說明
        • Netfilter - Wikipedia
        • 理解 Linux 下的 Netfilter/iptables - 知乎
        • Netfilter framework
          • Netfilter framework 提供了 packet filtering, network address translation (NAT), 和 port translation 功能
          • 從網路界面卡進入本機的 packet 會 "流經 Netfilter" 往送往本機應用程式或送出網路介面卡，Netfilter 在此 "流動過程" 中提供了數個 hooks
          • kernel modules 可以向 Netfilter hooks 註冊 callback functions 來達成前述功能，複數 kernel module 提供了不同的功能
        • iptables
          • ip_tables, ip6_tables, arp_tables, 和 ebtables kernel modules 提供 packet filtering 功能，有鑑於它們共用了許多程式碼，共用的部份後來獨立成 x_tables kernel module (由 modinfo 可知，前 4 個 module 皆依賴於它)
          • iptables*, ip6tables*, arptables*, 和 ebtables* userspace 指令用來操作前述的 5 個 kernel module
        • nftables
          • nftables 是 iptables 的繼承者
          • nf_tables kernel modules 提供 packet filtering 功能 (由於 nftables 的特性，不需要像 iptables 拆成複數 kernel module)
          • nft userspace 指令用來操作 nf_tables kernel module
        • iptables-legacy (iptables 軟體包) v.s. iptables-nft (iptables-nft 軟體包) v.s. nftables (nftables 軟體包)
          • iptables: The two variants and their relationship with nftables | Red Hat Developer
          • iptables-legacy 和 iptables-nft 的指令使用方式完全相同
          • iptables 和 iptables-nft 主要的差別是: 前者的命令大多連結到 xtables-legacy-multi，而後者則大多連結到 xtables-nft-multi
          • iptables-legacy 的 filtering rules 是由 x_tables kernel module 管理，nftables 的 filtering rules 則是由 nf_tables kernel module 管理，因此雖然他們同時都可以向 Netfilter 註冊 callback function，但使用 iptables-legacy 和 nft 指令卻無法看到對方的 filtering rules (TBD: 缺乏資料佐證，可能要翻程式碼看 rules 儲存在哪裡，但應該是這樣沒錯)
          • iptables-nft 可以想像成是一個 "指令翻譯器"，其 backend 為 nf_tables kernel module，因此 filtering rules 亦由它管理，當然使用 iptables-nft 和 nft 看到的 filtering rules 完全相同，看到的 filtering rules 和 iptables-legacy 完全不同 (TBD: 同前一條)

      • 軟體包: mate-system-monitor
        • gnome-system-monitor 跟 mate-system-monitor 是同樣的東西，只是前者使用 GTK3 而後者使用 GTK2，GTK3 的標題列和工具列合併在一起，不符合我的喜好
        • 因為功能太弱，改用 plasma-systemmonitor
      • 設定
        • 修正應用程式選單圖示未出現: 執行 grep --invert-match 'OnlyShowIn=MATE' /usr/share/applications/mate-system-monitor.desktop > ~/.local/share/applications/mate-system-monitor.desktop，執行後若應用程式選單未出現，執行 kbuildsycoca5

    • bbswitch (關閉獨立 NVIDIA GPU 電源)

      • 軟體包: bbswitch
        • 在 Acer TravelMate P2 (TMP215-53G-73RB) 機器上測試，關閉獨立顯示卡並不會降低電源使用功率，故不安裝
      • 設定
        1. 新增 /etc/modprobe.d/bbswitch.conf，內容為 
          blacklist nouveau
blacklist nvidia
options bbswitch load_state=0 unload_state=1
        2. 新增 /etc/modules-load.d/bbswitch.conf，內容為 
          bbswitch
        3. 重啟後生效
        4. 參考資料：Bumblebee - ArchWiki

      • 軟體包: java-openjfx
        • Oracle 在 JDK 11 之後不再包含 JavaFX，之後只作為第三方函式庫，因此只會越來越少人用，故不安裝
      • 說明
        • Arch Linux 把 JavaFX SDK 直接放在 Java 的資料夾 (/usr/lib/jvm/default/lib) 裡，跟 JRE 產生衝突，因此 JavaFX 程式無法正常執行。
