    Arch Linux 01 系統安裝

    參考資料

    • Arch Linux
    • Installation guide - ArchWiki
    • General recommendations - ArchWiki

    安裝規劃

    在此篇文章中，同時會包含以下系統安裝方式與設定。

    • 實體機和虛擬機 (VirtualBox)
    • 單系統 (Arch Linux) 和雙系統 (Windows 10 + Arch Linux)

    為使安裝步驟更通用，文章中會做出部分假設和簡化：

    • 命令和程式碼中帶有 <> 的部分應替換成實際內容
    • 只分割最少需要的分割區，若需 /home 或其他分割區，請自行修改安裝步驟
    • 假設只有一顆硬碟 (/dev/sda)

    另外，能夠在新系統安裝的部份就在新系統中安裝，減少使用 LiveCD 的時間。

    實體機

    需支援 UEFI 開機。

    虛擬機

    安裝至 VirtualBox 作為 Guest。機器設定:

    • 勾選 "啟用 EFI" 及 "硬體時鐘以 UTC 時間"
    • 圖形控制器使用 VMSVGA (如果使用 VBoxSVGA 有時候 host 和 guest 會一起當掉，只能在 host 切換到 VT kill VirtualBox，原因不明)
    • 硬碟控制器改成 "NVMe (PCIe)"，硬碟勾選 "固態磁碟機"

    VirtualBox 的 "設定 > 系統 > 開機順序" 是 BIOS 才有效，UEFI 有它自己的開機選單 (進入 UEFI 開機選單的方法：在啟動虛擬機器後立刻按下 Esc 鍵)。

    單系統

    UEFI 標準要求須要有至少一個 EFI System Partition (ESP)，並且 ESP 必須是 FAT32。

    雙系統

    測試過程見 "雜項" 一文。

    先安裝 Windows 10 (硬碟預留後面一塊空間)，再安裝 Arch Linux，可避免已經安裝好的資料被覆蓋掉 (特別是 boot loader 和 EFI boot entry)。

    由於 Windows Boot Loader 無法啟動 Windows，故需要另一個 boot loader 啟動 Linux。這裡使用 systemd-boot 作為 Arch Linux 的 boot loader，因為它的配置很簡單。

    為了開機可以進入 systemd-boot，需要把它加入 EFI boot entry，這點在執行 bootctl install 時就會完成。另外，在網路上有其他方法可以讓 Windows Boot Loader chain-load systemd-boot，但因為此方法需要修改到 Windows 的設定，故不採用。

    由於 kernel、initramfs 和 CPU microcode 太大 (systemd-boot 約 300 KB + kernel 約 11 MB + initramfs 約 42 MB + Intel microcode 約 5 MB)，放不進 Windows 建立的 ESP (只有約 100 MB)，故需要另外劃分一個分割區來放。該分割區需依照 systemd-boot 的要求指定 partition type 為 "Extended Boot Loader Partition (XBOOTLDR)" 才會被讀取。

    原先
ESP | Windows | Windows RE | Data
^
Windows BCD

再分割
ESP | Windows | Windows RE | XBOOTLDR | Linux root
^
Windows BCD & systemd-boot

    • systemd-boot Cannot launch binaries from partitions other than the ESP or the Extended Boot Loader Partition (XBOOTLDR partition).

      • Arch boot process - ArchWiki
      • systemd-boot - ArchWiki

    • UEFI may skip loading partitions other than the ESP when a "fast boot" mode is enabled. This can lead to systemd-boot failing to find entries on the XBOOTLDR partition; in that case, disable the "fast boot" mode.

      • systemd-boot - ArchWiki

    • The XBOOTLDR partition must be on the same physical disk as the ESP for systemd-boot to recognize it.

      • systemd-boot - ArchWiki

    安裝前準備

    • 製作 Arch Linux 安裝媒體

      • 下載映像檔：Arch Linux - Downloads
      • (僅實體機) 寫入 USB 隨身碟：USB flash installation medium - ArchWiki
        • 推薦使用 Ventoy (跨平台，Arch Linux 有 AUR ventoy-bin 可安裝) (見 "備註" 一節)

    • (僅實體機) UEFI BIOS 設定 (以 InsydeH2O® UEFI BIOS 為例)

      • Main > F12 Boot Menu: "Enabled" (systemd-boot 出問題時，可以啟動 Windows Boot Loader)
      • Main > Fast Boot: "Disabled" (開啟 Fast Boot 可能會導致 systemd-boot 無法讀取到 Extended Boot Loader Partition)
      • Boot > Boot Mode: "UEFI" (如果已經安裝了 Windows 10，應該預設就是 "UEFI")
      • Boot > Secure Boot: "Disabled" (關閉 Secure Boot 才能執行未簽章的 Boot Loader)

    • (僅實體機) 到電腦的 UEFI Boot Manager 設定以該 USB 隨身碟或光碟開機

    使用 LiveCD 將系統安裝至硬碟

    以下指令於 LiveCD 中執行。

    啟動安裝光碟

    1. 開機後會自動以 root 登入。
    2. 預設 keyboard layout 為 US，需要的話進行調整。
    3. 確認是否以 UEFI 開機：執行 [ -d /sys/firmware/efi/efivars ] && echo "booted in UEFI mode" || echo "booted in BIOS (or CSM) mode"。
    4. 連線到網際網路。
    5. 更新系統時間：執行 timedatectl set-ntp true。

    準備硬碟

    使用 cfdisk 進行分割 (見 "備註" 一節)。

    • 若硬碟沒有分割表，需先建立 GPT 分割表 (如果要刪除分割表，要用 fdisk，執行後按 g) (見 "備註" 一節)。

    • 分割 (單系統)

      Partition    Mount    Partition Type                  FS       Comment
/dev/sda1    /boot    EFI System                      FAT32    約 350 MB 即可 (見 "安裝規劃" 段落)
/dev/sda2    /        Linux filesystem                Ext4

    • 分割 (雙系統)

      Partition    Mount    Partition Type                  FS       Comment
/dev/sda1    /efi     EFI System                      FAT32    只有約 100 MB
/dev/sda2             Microsoft Reserved Partition    
/dev/sda3    C:       Microsoft Basic Data            NTFS
/dev/sda4             Windows Recovery Environment    NTFS
/dev/sda5    /boot    Linux extended boot             FAT32    約 350 MB 即可 (見 "安裝規劃" 段落)
/dev/sda6    /        Linux filesystem                Ext4

    • 格式化

      • Ext4: mkfs.ext4 <partition>
      • FAT32: mkfs.fat -F32 <partition>

    • 掛載 (以單系統為例)

      mount /dev/sda2 /mnt -o discard         # `-o discard` 是為了開啟 SSD TRIM (見 "備註" 一節)
mkdir /mnt/boot
mount /dev/sda1 /mnt/boot -o discard

    安裝最小可用系統

    • 安裝最小可用系統: 執行 pacstrap /mnt base linux linux-firmware (見 "軟體包清單" 一文)
    • 產生 fstab: 執行 genfstab -U /mnt >> /mnt/etc/fstab

    設定新系統運行的必要設定

    進入 chroot

    在 LiveCD 中執行 arch-chroot /mnt chroot 進新安裝的系統。

    以下指令於 chroot 進的新系統中執行。

    安裝設定所需的工具

    • 文字編輯器: 安裝 neovim 軟體包 (見 "軟體包清單" 一文)
      • 系統安裝結束後，可以刪除 /root/.local/share/nvim/shada/main.shada 和 /root/.cache/nvim/log

    基本系統設定

    • 設定系統時間
      • 執行 ln -sf /usr/share/zoneinfo/Asia/Taipei /etc/localtime
      • 執行 timedatectl set-ntp true
    • 設定語言及地區
      • uncomment /etc/locale.gen 裡的 en_US.UTF-8 UTF-8 和 zh_TW.UTF-8 UTF-8，儲存後執行 locale-gen
      • 在 /etc/locale.conf 中寫入 LANG=en_US.UTF-8
      • 如果前面的步驟中有修改 keyboard layout，編輯 /etc/vconsole.conf
    • 設定網路
      • 在 /etc/hostname 中寫入 hostname
      • 在 /etc/hosts 中寫入 
        127.0.0.1    localhost
::1          localhost
      • 安裝 networkmanager 並執行 systemctl enable NetworkManager (見 "軟體包清單" 一文)
    • 建立 Initramfs
      • 執行 mkinitcpio -P
        • https://wiki.archlinux.org/index.php/Mkinitcpio#Possibly_missing_firmware_for_module_XXXX
          wd719x 和 aic94xx 是 SCSI 的 firmware，一般電腦沒有。xhci_pci 是 USB 3.0 的 firmware (它改名叫 xhci-pci，所以找不到)
    • 設定 root 密碼: 執行 passwd

    安裝 Boot Loader

    測試過程見 "雜項" 一文。

    • systemd-boot 已經包含在 Arch Linux 的 systemd 軟體包裡，不用另外安裝。
    • 安裝 efibootmgr 軟體包 (見 "軟體包清單" 一文)
    • 安裝 intel-ucode 軟體包 (見 "軟體包清單" 一文)

    如果曾安裝過其它 UEFI Boot Loader，刪除舊的 EFI boot entry: 執行 efibootmgr -b <boot entry number> -B。

    • 單系統
      • 安裝 systemd-boot: 執行 bootctl install
      • systemd-boot 設定檔位置：/boot/loader/loader.conf
      • boot entry 設定檔位置：/boot/loader/entries/<boot entry>.conf
    • 雙系統
      • 安裝 systemd-boot: 執行 bootctl --esp-path=/efi --boot-path=/boot install
      • systemd-boot 設定檔位置：/efi/loader/loader.conf
      • boot entry 設定檔位置：/boot/loader/entries/<boot entry>.conf

    如果安裝成功，執行 efibootmgr -v 應該會看到 "Linux Boot Manager ... File(\EFI\systemd\systemd-bootx64.efi)"，代表 systemd-boot 已被加入 EFI boot entry，且 systemd-boot 會設為預設啟動項。但 Windows Boot Loader 還在，因此仍然可以使用它來啟動 Windows。

    同時，fallback boot manager (\EFI\BOOT\BOOTX64.EFI) 會被 systemd-boot 給覆蓋掉。如果需要復原，可以執行 cp /efi/EFI/Microsoft/Boot/bootmgfw.efi /efi/EFI/Boot/bootx64.efi (FAT32 不區分檔名大小寫)。

    安裝後設定 systemd-boot 並新增 boot entry

    • systemd-boot 設定

      # 以下 `default` 選項擇一
#default archlinux.conf    # 使用指定的 boot entry
#default @saved            # 使用上次選擇的項目 (見 `man loader.conf` "OPTIONS" 一節)
timeout 3

    • boot entry (以 archlinux.conf 為例)

      title Arch Linux
linux /vmlinuz-linux
initrd /intel-ucode.img
initrd /initramfs-linux.img
options root=UUID=<執行 `lsblk -f` 找出 UUID> rw nvme_core.default_ps_max_latency_us=0 ibt=off
# Linux kernel options 的 `nvme_core.default_ps_max_latency_us=0` 是為了修正 NVMe SSD 睡死問題  (見 "備註" 一節)
# Linux kernel options 的 `ibt=off` 是為了關閉 Intel IBT (見 "Linux 因為 Intel IBT 導致無法載入 module 問題" 一文)

      如果安裝了 linux-lts 或其他 kernel，記得也為它新增一個 boot entry。

      開機選單的項目順序是按照檔案名稱 (/boot/loader/entries/*.conf) 的字典序由下而上排列，排序結果會顯示於 bootctl list (見 "軟體包清單" 中 "systemd-boot" 一節)，可以使用 99-*.conf, 98-*.conf, 97-*.conf, ... 為檔名進行排列。

      systemd-boot 會自動偵測 Windows 10，不需要手動新增 boot entry。

    執行 bootctl list 應該會看到上面加入的 boot entry，注意一下有沒有打錯字造成無法啟動。

    未來如果更新 systemd-boot，需要手動更新 boot manager (執行 bootctl update)，或者安裝 systemd-boot-pacman-hook@A 自動在更新 systemd-boot 後更新 boot manager (見 "軟體包清單" 一文)。

    參考資料: systemd-boot - ArchWiki

    其他問題

    安裝完仍然會進入到 Windows 10，但用 F12 boot menu 卻可以進入

    在這台電腦上，可能是 UEFI 會自動把預設改為 Windows Boot Manager 開機。

    執行 efibootmgr --bootnum <bootnum> --inactive 停用 Windows Boot Manager 即可。

    Dual boot with Windows - ArchWiki

    Arch Linux 和 Windows 10 的系統時間不同步 (相差時區)

    Arch Linux 的硬體時間預設是設定為 UTC (timedatectl 顯示 RTC in local TZ: no)，系統時間則由 systemd 加上時區；但 Windows 10 的硬體時間預設是設定為 localtime，所以兩者會相差時區。

    ArchWiki 建議將 Windows 10 調整為使用 UTC：在機碼 HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\TimeZoneInformation 底下新增一個 DWORD RealTimeIsUniversal，值為 1，不需要同步時間，直接重啟後生效。

    為何推薦使用 Ventory 製作 USB 安裝媒體？

    Ventoy 的優點如下：

    • 跨平台 (Windows 和 Linux，Arch Linux 有 AUR ventoy-bin 可安裝)
    • 開源
    • 更新 ISO 檔不需要一直反覆格式化，只要把 ISO 檔複製進 USB 隨身碟即可，然後刪除舊的 ISO 檔
    • 可以放很多個 ISO 檔，製作多重開機隨身碟
    • 剩餘空間還可以存其他檔案，作為一般隨身碟使用
    • 有很多其他的 Plugin 可使用

    安裝 Ventoy 到 USB 後，USB 會分割成兩個磁碟區：一個用來存放 ISO/WIM/IMG/VHD(x)/EFI 檔案，另一個用來存放 Ventoy 程式 (通常檔案管理程式不會顯示此磁碟區)。

    資料來源：USB flash installation medium - ArchWiki

    其他試過的製作工具：

    • Rufus - 輕鬆製作可開機的 USB 磁碟機：如果不需要多重開機且只在 Windows 製作，則推薦使用 (Rufus 不是跨平台)
    • dd, cat, cp 等直接寫入的指令：(1) 有些 ISO 檔用此方式會無法開機 (2) 隨身碟無法再存其他檔案 (因為分割表被覆蓋了) (3) 不是跨平台

    USB flash installation medium - ArchWiki

    為什麼用 cfdisk 分割？

    • util-linux 軟體包提供了 fdisk, cfdisk, sfdisk 三個操作分割表的工具。三者功能完全相同 (皆使用 util-linux-libs 提供的 libfdisk.so)，只差在使用方式：
      • fdisk: dialog-driven
      • cfdisk: 好用的終端機圖形界面 (curses-based)
      • sfdisk: 給腳本使用的 (script-oriented)
    • 用 parted 分割 GPT 會偷偷加上 partition name (是存在 GPT 裡的名字，不是各分割區自己的 label，後者可用 e2label 查看)，因此改用 fdisk/cfdisk
    • 可進行 SSD 4K 對齊 (見 "備註" 一節)

    GUID Partition Table (GPT)

    GPT 的每個分割區都有一個 "partition type GUID" 用來代表該分割區的類型。

    分割區的 type 只是一串 GUID，和分割區會被格式化成什麼 file system 無關。但 UEFI、Boot Loader、作業系統和部分軟體會使用 type 資訊，所以仍應使用正確的 type。

    Linux 只有 "Linux filesystem data" 一種 type，不管要格式化成 ext4、swap 還是其他 file system 都可以使用它。但 systemd 自行制定了一個 Discoverable Partitions Specification (DPS)，裡面指定了 "Root partition"、"/usr/ Partition"、"Extended Boot Loader Partition (XBOOTLDR)"、"Swap"、"Home Partition" 等 partition type。

    其中，systemd-boot 所需要的就是 "Extended Boot Loader Partition (XBOOTLDR)"，在 cfdisk 叫做 "Linux extended boot"。

    另外，UEFI 所需要的 EFI System Partition (ESP) 的 partition type 是 "EFI System partition"。

    以上的 partition type 都可以用 fdisk/cfdisk 進行設定。

    BIOS 與 UEFI

    新的電腦使用 UEFI 取代傳統的 BIOS，UEFI+GPT 分割表需要多分割一個「EFI 系統分割區」(UEFI+MBR 分割表就不需要)，網路建議 EFI 系統分割區至少 512 MB。

    參考資料：
    Partitioning - ArchWiki
    EFI system partition - ArchWiki

    SSD 4K 對齊

    SSD 進行 4K 對齊 (即每個分割區的開頭都是 4 KiB 或 8 sectors 的倍數) 可提高效率。

    較新的磁碟分割程式 (如: fdisk/cfdisk, gparted) 都自動會做對齊 (不管是機械硬碟還是 SSD 都會對齊)。

    fdisk/cfdisk 會自動進行對齊，但因為不明原因它進行的是 1MiB 對齊，而不是 4KiB 對齊，所以如果分割區小於 1MiB，跟下一個分割區之間就會留空。

    SSD TRIM

    要開啟 TRIM 避免消耗，在磁碟機 mount 選項中加入 discard。

    NVMe SSD 睡死問題

    新的電腦使用 NVMe SSD 取代傳統的機械硬碟，原廠電腦有一個硬碟，後來又加購一個。

    但在使用時發現，如果將 /home 放在第二個硬碟上，並執行 I/O 頻繁的程式 (如：在 VirtualBox guest 裡執行 Pacman 系統升級，或在 host 安裝 Mathematica)， host 和 guest 就會直接卡住（但執行中的程式只要不讀取硬碟就可以繼續執行），只能強制重開機。

    從 journalctl 有發現 kernel: nvme nvme1: I/O 320 QID 4 timeout, aborting(數字可能不同)，確定問題出在 NVMe SSD 上。

    根據網路資料，確認過不是溫度太高、RAM 用完造成，也更換過一顆新的硬碟，所以問題出在 Linux 上 (因為沒發現 Windows 有這個問題)。

    因為可能是 I/O 太頻繁的關係，但之前往硬碟寫了 200 多 GB 的資料也沒事，所以也有可能和 VirtualBox 有關。 根據網路資料，(1) System: 打勾 "Enable I/O APIC" (2) SATA controller: 打勾 "Use host I/O cache" (3) VDI: 打勾 "Solid-state drive"，無效。

    根據網路資料，可能是 Linux 的 NVMe 驅動程式中的省電模式有問題，導致硬碟進入省電模式後就無法恢復(睡死)，關掉省電模式就可以解決。

    在 Linux kernel options 加上 nvme_core.default_ps_max_latency_us=0:

    • GRUB: 編輯 /etc/default/grub，在 GRUB_CMDLINE_LINUX_DEFAULT 加上 nvme_core.default_ps_max_latency_us=0，儲存後執行 update-grub。
    • systemd-boot: 編輯 /efi/loader/entries/*.conf，在 options 加上 nvme_core.default_ps_max_latency_us=0，儲存後即可。

    cat /sys/module/nvme_core/parameters/default_ps_max_latency_us 原本 100000，修改後為 0。

    經過長期測試發現有效。
